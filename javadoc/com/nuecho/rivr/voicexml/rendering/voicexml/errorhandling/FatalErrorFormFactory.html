com.nuecho.rivr.voicexml.rendering.voicexml.errorhandling

Interface FatalErrorFormFactory

  • All Known Implementing Classes:
    ExitFatalErrorFormFactory, ReturnFatalErrorFormFactory

    public interface FatalErrorFormFactory
    Generates part of the VoiceXML document responsible for handling of VoiceXML fatal errors (error events that can't be handle by the HTTP server).

    Normally, VoiceXML error events are sent back to the dialogue for processing. However, if another error occurs when the first error is sent back to the server, we have a case of fatal error and must have an error handler in the VoiceXML to deal with these errors. This class contains the addFatalErrorForm(Element, VoiceXmlDocumentTurn) method which is invoked to add the proper VoiceXML elements in order to deal properly with such errors.

    By default, the VoiceXmlDialogueServlet is initialized with a ExitFatalErrorFormFactory, which generates an empty <exit> element. This is generally acceptable. However, some contexts would require a different handler, such as subdialogues where a <return> would be more appropriate.
